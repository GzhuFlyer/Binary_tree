从Hello world 到二叉树（C 语言）
-----
琐碎的话：对于一个编程人员来说，从入手一门编程打印出一句Hello World!,到最后使用编程语言得心应手的开发出各种软件，似乎没有人能真正了解打印出Hello world语句背后都经历些什么。比方说当我们写下了Hello World 代码到编译器编译生成可执行文件都发生了些什么，可执行文件在操作系统上运行操作系统进行了那些工作，编译器也是一个软件，一开始是什么样子，总不该突然就有编译器了吧。操作系统是用什么编程语言编写的，计算机一开始可是一个没有屏幕没有键盘的东西，只能像单片机一样用来点亮一个Led灯而已，从一开始发展到现在，我们难道还会用高低电平来描述分析计算机运行过程中发生了些什么。总而言之，Hello World语句后面包含了很多很多知识，对于这些知识的了解不用面面俱到也可以使我们开发出各种各样的软件供我们使用。
	为什么我们可以放心的开发和使用我们的软件呢？我想这里涉及到一个很有趣的东西。那就是基于假设上的推理和结论的验证。这是很有趣的东西，比方说，对于数学上简单的加减乘除，1+1=2，我想没有人能够给出证明（当然如果有人说这是公理，因而不用证明，我也没话可说），我们仅仅是给出1到100这些数字并告诉加法这个东西怎么用，人们便能轻易计算出任意两个1到100之间的数字是怎样去使用，甚至能够创造出更多的数字，将更大的数字进行相加。最后，当我们通过自身的思考将两个数字相加后得到另一个数的时候，我们怎么知道自己计算出来的这个数是正确的呢？这里就涉及到验证，怎么去验证？这里给出了一个很简单的方法，比方说12+13=25，我们想验证25这个数值，我们可以将12根火柴和13根火柴放在一起（因为放在一起实际上就是一个加的操作），再从头开始数，数完所有放在一起的火柴，得出来的结论就是25了，当然这里我们没有任何兴奋和成就感，毕竟这算不上完成一个惊人的项目，但是这里却是闪耀着人类智慧的光芒，从一个已有的东西推理出新的东西，再到验证出这个东西可以用，这是何等的美妙。
